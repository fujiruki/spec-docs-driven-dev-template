対象プロジェクトに仕様書駆動開発（SdDD）の環境をセットアップします。

## 手順

### Step 1: 対象プロジェクトの確認

発注者に対象プロジェクトのパスを確認する。
カレントディレクトリが対象なら「このディレクトリにセットアップしますか？」と聞く。

### Step 2: 現状スキャン

対象プロジェクトの以下を調べて報告する:

- `CLAUDE.md` が既に存在するか
- `docs/` ディレクトリが存在するか
- `.claude/commands/` が存在するか
- `task.md` が存在するか
- プロジェクト内の既存ドキュメント（`.md` ファイル、README等）

報告例:
```
対象: /path/to/project
- CLAUDE.md: あり（既存の内容あり）
- docs/: なし
- .claude/commands/: なし
- 既存ドキュメント: README.md, docs/旧仕様.md（2件）
```

### Step 3: 発注者に個別に確認

以下を**個別に**確認する。まとめて聞かない:

1. **CLAUDE.md について**
   - 既存あり → 「既存のCLAUDE.mdがあります。SdDDルールを追記しますか？それとも新規に作り直しますか？」
   - 既存なし → 「CLAUDE.mdを新規作成します」と報告

2. **既存ドキュメントについて**（ある場合のみ）
   - 「既存のドキュメントが見つかりました。これを元に仕様書（docs/spec/）にまとめましょうか？新しいものほど重要として扱います」
   - Yes → Step 5 で取り込み処理を行う
   - No → 空のテンプレートを配置

3. **スラッシュコマンドについて**
   - 「SdDD用のスラッシュコマンド（/spec-sync, /debug, /kaigi 等）を配置しますか？」

### Step 4: ファイル配置

発注者の回答に応じてファイルを配置する。
テンプレートのソースは、このリポジトリの `templates/` ディレクトリから取得する。

配置するもの:
- `CLAUDE.md` — 追記 or 新規作成（Step 3の回答に従う）
- `docs/SPEC.md` — 仕様書の目次
- `docs/requests.md` — 要望入力口
- `docs/request_log.md` — 要望対応履歴
- `docs/spec/` — 仕様書テンプレート群（01-06）
- `docs/handover/` — Agent引き継ぎ資料フォルダ
- `task.md` — タスク管理ファイル
- `.claude/commands/` — スラッシュコマンド群（Step 3で承認された場合）

### Step 5: 既存ドキュメントの取り込み（該当する場合のみ）

発注者が取り込みを希望した場合:

1. 既存ドキュメントをファイル更新日時の新しい順に読む
2. 内容を分類して対応する spec ファイルに反映:
   - プロジェクトの目的・概要 → `spec/01_概要.md`
   - 機能に関する記述 → `spec/02_機能仕様.md`
   - 画面・UIに関する記述 → `spec/03_画面設計.md`
   - DB・APIに関する記述 → `spec/04_データ設計.md`
   - 技術スタック・構成 → `spec/05_技術設計.md`
3. 取り込み元と取り込み内容を `spec/06_変更履歴.md` に記録
4. `SPEC.md` の目次を更新
5. 内容が矛盾する場合は新しいドキュメントを優先し、発注者に確認する

### Step 6: ダッシュボードの案内

```
進捗ダッシュボード（任意）を使うと、Agentの作業状況をリアルタイムで確認できます。

セットアップ:
  cd <spec-docs-driven-dev-templateのパス>/dashboard
  npm install && npm run build && npm link

使い方（別ターミナルで実行）:
  sdd-dashboard <プロジェクトルートの絶対パス>/task.md
```

### Step 7: 完了報告

セットアップ内容をまとめて報告し、次のステップを案内:

```
セットアップ完了しました。

配置したもの:
- CLAUDE.md（SdDDルール追記）
- docs/（仕様書テンプレート一式）
- .claude/commands/（スラッシュコマンド12個）
- task.md（タスク管理）

次のステップ:
- /sddd で開発を始められます
- docs/requests.md に要望を書くか、直接会話で伝えてください
```

## 注意

- 発注者の許可なくファイルを上書きしない
- 既存ファイルを削除しない
- セットアップ中に実装作業を開始しない
